# Промт для Replit AI — FitPro Platform (MVP)

> Скопируй весь текст ниже в Replit AI Agent. Он описывает MVP целиком: стек, модель данных, роли, экраны и поведение. Если Replit предложит разбить на этапы — соглашайся, порядок этапов указан в конце.

---

## ЗАДАЧА

Создай веб-приложение **FitPro Platform** — «операционную систему для онлайн-тренера». Это SaaS, который заменяет тренеру Telegram, таблицы, заметки, PDF и платёжные ссылки единым пространством для ведения клиентов. Сделай работающий MVP (не макет): с реальной базой данных, авторизацией, двумя ролями (тренер и клиент) и полным CRUD по основным сущностям.

Интерфейс полностью на **русском языке**. Дизайн — чистый, современный, мобильно-адаптивный (тренер и клиент часто работают с телефона).

---

## ТЕХНИЧЕСКИЙ СТЕК

- **Frontend:** React + TypeScript, Vite, Tailwind CSS, React Router. Компоненты — shadcn/ui (или аналог). Иконки — lucide-react. Графики — recharts.
- **Backend:** Node.js + Express (TypeScript).
- **БД:** PostgreSQL через Drizzle ORM (или Prisma). Используй встроенную в Replit БД.
- **Авторизация:** email + пароль, сессии через JWT в httpOnly cookie. Роли: `trainer`, `client`.
- **Файлы:** загрузка фото/видео/PDF в объектное хранилище Replit (или локально с отдачей по URL). В карточках хранить ссылки.
- Сделай seed-скрипт с демо-данными: 1 тренер + 4–5 клиентов с заполненными анкетами, программами, отчётами, замерами.

---

## РОЛИ И ДОСТУП

**Тренер** видит всех своих клиентов, создаёт программы, упражнения, отчёты-формы, материалы, ставит задачи, видит аналитику и финансы.
**Клиент** видит только свой кабинет: назначенные тренировки, дневник, отчёты к заполнению, задачи/привычки, материалы (с поэтапным доступом), свой прогресс.

После логина роль определяет, какой дашборд показывается.

---

## МОДЕЛЬ ДАННЫХ (минимум)

- **User** — id, email, passwordHash, role, name.
- **Client** — привязан к тренеру и к user-аккаунту клиента. Поля: имя, возраст, рост, вес, цель, уровень, формат работы, датаСтарта, срокСопровождения(дата окончания), статус воронки.
- **ClientProfile (анкета)** — опыт тренировок, травмы/ограничения, образ жизни (работа, сон, стресс), питание, шаги, оборудование, предпочтения/антипредпочтения.
- **TrainerNote** — свободные заметки тренера по клиенту (несколько записей с датой).
- **Exercise (библиотека упражнений)** — название, видео-URL, описание техники, ключевые подсказки, типичные ошибки, мышцы, варианты упрощения/усложнения. Принадлежит тренеру.
- **WorkoutTemplate (шаблон программы)** — название, цель (похудение/масса/рекомпозиция/дом/новичок/продвинутый), список упражнений с параметрами.
- **Workout (назначенная тренировка)** — клиент, дата, список упражнений с подходами/повторениями/весом/темпом/отдыхом/комментарием.
- **WorkoutLog (дневник)** — по каждому упражнению: подход, вес(кг), повторения, ощущения (легко/умеренно/тяжело/очень тяжело). Заполняет клиент.
- **Measurement (замеры)** — дата, вес, талия, бёдра, грудь, рабочий вес по ключевым упражнениям + фото-URL «до/после».
- **ReportForm + ReportSubmission** — настраиваемая форма еженедельного отчёта (поля: вес, шаги, сон, энергия, настроение, соблюдение питания, срывы, выполнение тренировок, что было сложно, фото, свободный комментарий) и заполнения клиента со статусом (ожидает проверки / проверен / пропущен).
- **Task/Habit** — задачи на неделю (10000 шагов, вода, урок, белок, сон до 23:00, зарядка, тренировка). Клиент отмечает выполнение, считается % соблюдения.
- **KnowledgeItem (база знаний)** — категория (Питание/Тренировки/Замеры/Восстановление), название, тип (PDF/видео/чек-лист), файл-URL, неделя открытия (для поэтапного доступа).
- **Payment** — клиент, сумма, дата, статус (оплачено/просрочено), дата следующего продления.
- **Achievement** — бейджи (10 тренировок, месяц без пропусков, минус 5 см, рост рабочего веса), streak (недель без пропусков).

## ЭКРАНЫ ТРЕНЕРА

### 1. Главный экран (панель управления бизнесом)
Не лента сообщений, а оперативная сводка. Виджеты-карточки со счётчиками и списками клиентов, по которым нужно действие:
- Новые заявки и анкеты (ожидают обработки)
- Ожидают проверки отчёта
- Не отправили отчёт (пропустили срок)
- Пропустили тренировку (нет активности)
- **Зона риска** — 7+ дней без активности (метка ставится автоматически)
- Заканчивается сопровождение (до окончания < 7 дней)
- Просроченная оплата
- Лучший прогресс недели
Цель: за 10 секунд понять, что происходит с каждым клиентом. Клик по виджету ведёт к отфильтрованному списку.

### 2. CRM / список клиентов
Воронка по статусам: **Новая заявка → Анкета заполнена → Созвон → Ожидает оплату → Активный → Заморожен → Заканчивает → Архив**. Канбан или таблица с фильтром по статусу. Цветовые метки риска.

### 3. Карточка клиента (сердце приложения)
Вкладки: **Основные данные · Анкета · Заметки тренера · История работы (оплаты, прошлые программы, отчёты, замеры, фото) · Тренировки · Прогресс/Аналитика**.

### 4. Конструктор тренировок
- Библиотека упражнений (CRUD, с видео и заметками тренера).
- Сборка тренировки из упражнений (подходы/повторения/вес/темп/отдых/комментарий).
- Шаблоны программ по целям + адаптация под клиента.
- Назначение программы клиенту в один клик; история в карточке.

### 5. Контроль прогресса и аналитика (на клиента)
Графики recharts: прогрессия рабочих весов по упражнениям, посещаемость/регулярность, выполнение плана в %, субъективная тяжесть в динамике, выявление плато. Таблица замеров с изменениями (вес/талия/бёдра/рабочий вес, +/− и цветом). Таймлайн фото-трансформации (сравнение периодов).

### 6. Отчёты
Конструктор формы отчёта. Список заполнений по статусам (кто отправил / ждёт проверки / пропустил). Просмотр и пометка «проверен».

### 7. Задачи и привычки
Назначение недельных задач клиенту, просмотр % соблюдения.

### 8. База знаний
Загрузка материалов по категориям + настройка поэтапного открытия по неделям.

### 9. (Базово) Финансы
История оплат, статус (оплачено/просрочено), напоминание клиенту, дата продления. Без реального эквайринга — статусы и история вручную/демо.

---

## ЭКРАНЫ КЛИЕНТА (личный кабинет)

- **Главная:** сегодняшняя/ближайшая тренировка, незаполненный отчёт, задачи на сегодня, прогресс-бар месяца, streak и бейджи.
- **Тренировки:** список назначенных; внутри — **дневник**: по каждому упражнению вводит подход/вес/повторения/ощущения, отмечает выполнение. Видит видео и подсказки именно своего тренера.
- **Прогресс:** свои графики и таймлайн фото, ввод замеров (с напоминанием по периодичности).
- **Отчёты:** заполнение еженедельной формы.
- **Задачи/привычки:** чек-лист, отметка выполнения.
- **Материалы:** база знаний с поэтапным доступом (открыто только то, что положено по текущей неделе сопровождения).

---

## ЛОГИКА И АВТОМАТИЗАЦИЯ (важно для MVP)

- **Авто-метка «зона риска»** при 7+ днях без активности клиента.
- **Авто-статусы отчётов:** «пропущен», если не отправлен к дедлайну; «ожидает проверки» после отправки.
- **Расчёт % соблюдения** задач и выполнения плана тренировок.
- **Уведомления (внутри приложения, колокольчик):** новый отчёт, пропуск тренировки, заканчивается сопровождение, просрочена оплата — для тренера; новая программа, напоминание об отчёте/замерах/задачах — для клиента.
- **Геймификация:** начисление бейджей и streak по событиям.

## ТАРИФЫ (статическая страница/настройка, без оплаты в MVP)
Базовый ~990 ₽/мес (до 10 клиентов), Профессиональный ~2490 ₽/мес (до 50), Экспертный ~4990 ₽/мес (до 100), Студия — индивидуально. Покажи на лендинге/в настройках; ограничение по числу клиентов можно отразить как мягкое предупреждение.

## ЧЕГО НЕ ДЕЛАТЬ В MVP (заглушки/следующие этапы)
AI-помощник, реальный эквайринг и автосписание, маркетплейс тренеров, интеграции с Apple Health/Garmin/Whoop, командные роли (куратор/менеджер), белый лейбл. Оставь под них кнопки/разделы с пометкой «скоро».

---

## ПОРЯДОК РЕАЛИЗАЦИИ (этапы)
1. Каркас проекта, БД, схема, авторизация и роли, layout с навигацией.
2. CRM: клиенты, статусы-воронка, карточка клиента + анкета + заметки.
3. Упражнения, конструктор тренировок, шаблоны, назначение клиенту.
4. Кабинет клиента + тренировочный дневник.
5. Замеры, фото, аналитика и графики прогресса.
6. Отчёты (форма + заполнение + статусы), задачи/привычки, геймификация.
7. База знаний с поэтапным доступом.
8. Главный дашборд тренера со всеми виджетами + уведомления + базовые финансы.
9. Seed-данные, проверка сквозного сценария «тренер ↔ клиент», адаптив.

После каждого этапа показывай работающий результат, прежде чем идти дальше.

---

## КРИТЕРИЙ ГОТОВНОСТИ MVP
Тренер заводит клиента → клиент заполняет анкету → тренер собирает тренировку и назначает → клиент выполняет и пишет в дневник → клиент сдаёт замеры/фото/отчёт → тренер видит прогресс на графиках и сводку на главном экране, включая авто-метку риска. Всё в одном приложении, без сторонних сервисов.
